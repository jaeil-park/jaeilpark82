* **기술/도구:** GitHub, Bash Script, Markdown, Mermaid.js
* **분석:** C님은 단순 이력만 나열하지 않고, 실무에서 서버 세팅을 자동화할 때 썼던 `쉘 스크립트(Bash)`들을 모아 Repository로 만들었습니다. 또한, Pre-Sales 제안서에 들어가는 아키텍처 다이어그램을 Markdown(Mermaid.js)으로 깔끔하게 그려 README에 첨부했습니다. 채용 담당자는 이를 보고 "이 사람은 문서를 체계적으로 다룰 줄 알고, 업무 효율을 높이는 진짜 엔지니어"라고 평가했습니다.

**❌ 실패 사례: 코드 없는 텅 빈 프로필 (인프라 엔지니어 D님)**
* **기술/도구:** GitHub (계정만 존재)
* **분석:** D님은 최신 트렌드를 따라 GitHub을 만들었지만, 개발자가 아니라는 이유로 소스 코드를 올리지 않고 README에 이력서 텍스트만 복사해 두었습니다. 이는 기존 Notion이나 PDF 이력서와 차별점이 없었으며, 오히려 GitHub의 기능을 활용하지 못해 디지털 리터러시가 부족해 보이는 역효과를 낳았습니다.
